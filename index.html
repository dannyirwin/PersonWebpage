Sightless VR is a completely auditory and haptic VR expeience for Occulus. I was approached by my good friend and develompent mentor who had the initial idea. The experience is fairly simple: navigate throuhg a 3D space using only the sounds around you and the haptic vibration from the controllers to get your barings. In the space, there are randomly place obsticles, enemies on patrol who will give chace, and mines scattered about. It was built using Unreal Engine and currently under review on the Occulus store

This was my first experience developing with somebody else wich proved to be extremely educational. I was able to learn about version and source managment on a team, readability and documetation, and collaberative problem solving. It was also invaluable to see how a professional might work through the entire devolment process from initial concept meetings to publishing. Our personal goals with this project was to see how well we can work together, experiment with visualless feedback in VR, and to see the project all the way through publication.

Along with working on the mines and enemies, my main resposiblity was the sound and music managers, as well as composing and producing the music itself. One key feature which provided interesting challege for me was the dynamic music. As an indication of how far the player is to the exit, the music itslef changes to more or less complicated melodies. It was a lovely oportunity to use music theory and compositon tricks to solve programatic issues, and visa versa.

The Fame Game is my run at building a web-based idle clicker game. The basic premise is that the player is attempting to accumulate "Fame" by slowing building up diferent kinds of followers and fans. Each type of follower generates "Fame" which serves as a currency to purcahse more followers which then compound.

The project was built only using HTML, CSS, JavaScript, and Jquery. One major development goal of the project was to be able to easily add and remove diferent followers and having all of their UI elements be generated programatically. One of the features is a custom follower tab where the user can input the title and properties of a new follower and have it appear in the game with all the necisary button and display information. Another problem I found interestng to solve was converting the very large numbers to shorter notation to fit in the UI elements. For example, it would display a number like 10 Billion--or 10,000,000,000.00--to 10.00B using log10 and other arithmatic functions. This was also my introduction of using CSS Flexbox to keep the follower UI elements orgaized in a pleasing way as elemets were added, subrtracted, and sorted.

This project is still in development.

This Driving Schelue Generator was my first project outside the ones assigned by the FreeCodeCamp courses I used to learn web development. I had been hired as an actor on a national childrns' theatre tour where the entire company, incluidng actors, were responsible for driving a set van and passanger van all over the country. We used this app to schedlue hundreds of hours and thougsands of miles of driving.

This app was designed to use a variety of rules to generate a fair and balanced driving schedule. These rules took into account how often somebody had driven a vehicle, who had worked as a navigator, building in rest shifts, and pairing drivers and navigators evenly. I achieved this by assigning each rule a numerical weight and scoring the most viable permutations against each other. Troubleshooting the schedule was difficult but very interesting as the patterns that developed were not always as expected and could wade into some pretty high-level math theory waters.
